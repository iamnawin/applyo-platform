# 🕵️ Diagnosis Report: Applyo Platform Issues

This document outlines the exact technical reasons why the Applyo platform is currently experiencing intermittent **504 Gateway Timeouts** and "non-clickable" behavior on Vercel.

## 1. The "Non-Interactive" Issue (Hydration Failure)
**Symptom**: You can see the page (buttons, text, images), but clicking "Get Started" or selecting a role does nothing.
**Technical Cause**:
Next.js uses a process called **Hydration**. It first sends a fast "Skeleton" of HTML (which you see), then it tries to download the JavaScript "Brain" (chunks) to make the page interactive.
*   **The Failure**: Because of the Middleware issue (see below), the request for the JavaScript chunks is being blocked or timed out by Vercel. 
*   **Result**: The page has the "Body" but no "Brain." No event listeners are attached, so clicks do nothing.

---

## 2. The 504 Gateway Timeout (Edge Middleware)
**Symptom**: In less than 1 minute (often exactly at 10-15 seconds), the page crashes with a `504` error.
**Technical Cause**: **Vercel Edge Middleware Execution Limits.**
Vercel's free and pro tiers have a very strict **50ms CPU limit** for "Edge" middleware. 
*   **The Problem**: Our `middleware.ts` was calling `supabase.auth.getUser()` for **every single request** (including requests for JS chunks, CSS, and images).
*   **The Latency**: If the database (Supabase) takes more than 50ms to respond for a user in India (due to the distance to the server in the US), Vercel **kills the request immediately**.
*   **The Chain Reaction**: When the middleware is killed, the JS chunks fail to load → The page fails to hydrate → The buttons stop working → A 504 error is shown.

## 3. Active Research & Logs
We have verified the following:
*   [x] **Build Success**: The backend is building correctly (no more bundle errors).
*   [x] **Regional Issue**: The site works for the AI Agent (US-based) but fails for the User (India-based). This confirms it is a **latency-based timeout**.
*   [x] **Middleware Bottleneck**: Found that `getUser()` is running on static assets, which is a known performance anti-pattern for Vercel.

---

## 🚀 The Plan to Fix It permanently
We are moving to a **"Lazy Auth"** strategy:
1.  **Strict Middleware**: The middleware will **ONLY** run for protected routes like `/dashboard`. It will ignore the Landing page, Signup, and Login pages entirely.
2.  **Asset Exemption**: No middleware will run for `.js`, `.css`, or images.
3.  **Page Resilience**: The pages will handle their own redirects. This runs in a "Serverless Function" (not Edge), which has a **10-second timeout** instead of 50ms. This is 200x more stable.
